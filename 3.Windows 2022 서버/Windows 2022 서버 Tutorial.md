###  1. IP 주소 (Internet Protocol Address)

**개념:** 네트워크 공간에서 내 서버가 가진 **고유한 식별 번호**입니다.

- **구조:** 0~255 사이의 숫자 4마디로 구성됩니다 (예: `192.168.0.10`).
- **역할:** 데이터를 보낼 때 '어디로' 보낼지, 혹은 '어디서' 왔는지를 나타내는 최종 목적지입니다.

### 2. 서브넷 마스크 (Subnet Mask)

**개념:** IP 주소에서 어디까지가 '우리 동네(네트워크)'이고, 어디부터가 '내 컴퓨터 번호(호스트)'인지 구분하는 절취선입니다.

- **형태:** 보통 `255.255.255.0`과 같은 형태를 띱니다.
- **역할:** 서버는 서브넷 마스크를 통해 목적지 IP가 같은 동네(로컬)에 있는지, 아니면 외부(인터넷)에 있는지 판단합니다.
    - 같은 동네면 직접 전달하고, 아니면 아래 설명할 '게이트웨이'로 보냅니다.

서브넷 마스크의 원리를 이해하려면 컴퓨터가 숫자를 바라보는 방식인 **'2진수(Binary)'**의 세계로 잠시 들어가야 합니다.

컴퓨터는 우리가 쓰는 `255.255.255.0`을 그대로 이해하지 않고, 32비트의 0과 1의 나열로 처리합니다. 왜 `255`가 나오는지, 그리고 어떻게 네트워크를 나누는지 상세히 설명해 드릴게요.

---

> **왜 하필 255인가? (8비트의 비밀)**
> 
> IP 주소 한 마디(Octet)는 **8개의 비트**로 이루어져 있습니다.
>  비트 하나는 0 또는 1만 가질 수 있습니다.
>   8개의 비트가 모두 1일 때($11111111_2$), 이를 10진수로 계산하면 **255**가 됩니다.
    - $2^7 + 2^6 + 2^5 + 2^4 + 2^3 + 2^2 + 2^1 + 2^0 = 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255$
	즉, `255.255.255.0`을 2진수로 풀어서 쓰면 다음과 같습니다.
    `11111111 . 11111111 . 11111111 . 00000000`

> **서브넷 마스크의 작동 구조: AND 연산**
> 
> 서브넷 마스크의 핵심 역할은 **"IP 주소에서 어디까지가 네트워크 이름표인가?"**를 결정하는 것입니다. 컴퓨터는 이를 위해 **논리곱(AND) 연산**을 수행합니다.
> **AND 연산 규칙:** 둘 다 1일 때만 결과가 1이 됨.
> - 만약 내 IP가 `192.168.0.10`이고, 마스크가 `255.255.255.0`이라면:

| **구분**              | **1마디**        | **2마디**        | **3마디**        | **4마디**        |
| ------------------- | -------------- | -------------- | -------------- | -------------- |
| **IP 주소 (2진수)**     | `11000000`     | `10101000`     | `00000000`     | `00001010`     |
| **마스크 (2진수)**       | `11111111`     | `11111111`     | `11111111`     | `00000000`     |
| **결과 (Network ID)** | **`11000000`** | **`10101000`** | **`00000000`** | **`00000000`** |
**결과값:** `192.168.0.0` 이 나옵니다. 이것이 이 서버가 속한 **네트워크 주소(대표 이름)**가 됩니다. 
- **해석:** 마스크의 1이 연속된 부분까지는 '동네 이름(Network)'이고, 0인 부분은 '그 동네의 번지수(Host)'가 됩니다.
    
> **왜 이렇게 복잡하게 구분할까?**
	서버가 데이터를 보낼 때, 목적지 IP와 자신의 서브넷 마스크를 연산해 봅니다.
	1. **결과가 내 네트워크 주소와 같다면:** "아, 같은 동네 사람이구나!" → 직접 전달 (L2 통신)  
	2. **결과가 내 네트워크 주소와 다르다면:** "다른 동네네? 게이트웨이(출입구)로 보내야겠다!" → 외부 전달 (L3 통신)

> **CIDR 표기법 (슬래시 뒤의 숫자)**
> 실무에서는 `255.255.255.0` 대신 **/24**라고 부르는 경우가 많습니다.
>  이는 왼쪽부터 **1이 연속된 개수**를 의미합니다. 
>  `11111111(8) . 11111111(8) . 11111111(8) . 00000000(0)` = 총 24개.  
>  그래서 `192.168.0.10/24`라고 쓰면 "서브넷 마스크가 255.255.255.0이구나"라고 이해하면 됩니다.

---

> **💡 서브넷 분할**
> 만약 `255.255.255.128`(/25)을 쓴다면 어떻게 될까요?
> 마지막 마디가 `10000000`이 됩니다. 이렇게 되면 하나의 네트워크를 반으로 쪼개서 관리할 수 있게 됩니다. 이를 **서브네팅(Subnetting)**이라고 하며, IP 자원을 효율적으로 나누고 보안 영역을 분리할 때 필수적으로 사용합니다.


### 3. 게이트웨이 (Gateway) 주소

**개념:** 우리 동네 네트워크에서 **외부(인터넷)로 나가는 '출입구'**의 주소입니다.
- **구조:** 보통 해당 네트워크의 첫 번째 주소(`192.168.0.1`)를 관례적으로 사용합니다.
- **역할:** 내 서버가 모르는 외부 IP 주소로 데이터를 보내려고 할 때, "난 모르겠으니 일단 출입구(게이트웨이)로 보낼게!"라고 하며 데이터를 던지는 곳입니다. 보통 공유기나 L3 스위치의 IP가 됩니다.

### 4. DNS 서버 주소 (Domain Name System)

**개념:** `naver.com` 같은 문자를 `223.130.200.104` 같은 **숫자 IP로 바꿔주는 '전화번호부'** 서비스입니다.
- **역할:** 우리는 숫자를 다 외울 수 없기 때문에, 브라우저에 주소를 치면 먼저 DNS 서버에 물어봅니다. "구글 IP 주소가 뭐야?"라고 물으면 DNS가 숫자를 알려주고, 서버는 그 숫자를 보고 접속합니다.
- **대표 주소:** 구글의 `8.8.8.8`이나 통신사(KT, SK 등)의 DNS를 주로 입력합니다.

---

> 💡 **실습 팁 (Windows Server 입력 시)**
	Windows 서버의 **[네트워크 및 공유 센터] -> [어댑터 설정 변경] -> [이더넷 속성] -> [IPv4]** 메뉴에서 이를 입력하게 될 텐데, 가상머신 환경(VMware, VirtualBox 등)에 따라 설정 방식이 달라집니다.
	**NAT 모드:** 가상 소프트웨어가 자동으로 할당해주는 대역을 써야 합니다. 
	**Bridge 모드:** 실제 공유기(공유기 IP 대역)와 같은 대역의 주소를 써야 합니다.


## 네트워크 카드
NIC, 랜카드, 네트워크 어댑터 등

네트워크 카드는 단순해 보이지만 내부적으로는 복잡한 처리를 담당하는 작은 컴퓨터와 같습니다.

- **물리적 포트 (RJ-45):** 랜선을 꽂는 구멍입니다. 데이터가 전기 신호로 드나드는 입구입니다.
    
- **컨트롤러 칩셋:** 데이터를 패킷으로 쪼개거나 합치고, 오류가 있는지 검사하는 두뇌 역할을 합니다 (Realtek, Intel 등이 유명합니다).
    
- **MAC 주소 (ROM):** 공장에서 찍혀 나오는 하드웨어 고유 번호입니다. "전 세계에서 유일한 주민등록번호"라고 생각하면 됩니다.
    
- **버스 인터페이스:** 메인보드(PCIe 슬롯 등)와 연결되어 CPU/메모리와 데이터를 주고받는 통로입니다.

## 3. 핵심 기능 (네트워크 카드가 하는 일)

네트워크 카드는 **OSI 7계층 중 1계층(물리)과 2계층(데이터 링크)**의 가교 역할을 합니다.

### ① 데이터 직렬화 및 병렬화 (Serialization)

컴퓨터 내부의 데이터는 병렬(여러 줄)로 흐르지만, 랜선은 직렬(한 줄)로 데이터를 보냅니다. NIC는 이 데이터를 한 줄로 세워 보내고, 받을 때는 다시 여러 줄로 합칩니다.

### ② MAC 주소 판별

네트워크에 흐르는 수많은 패킷 중 **"이게 내 주소(MAC)로 온 게 맞는가?"**를 확인합니다. 내 것이 아니면 CPU까지 전달하지 않고 NIC 선에서 바로 버려버립니다. (CPU의 부하를 줄여주는 중요한 기능입니다.)

### ③ 데이터 캡슐화 (Framing)

전송할 데이터 앞뒤에 '헤더'와 '트레일러'를 붙여 **프레임(Frame)**이라는 단위로 만듭니다. 여기에 출발지/목적지 MAC 주소와 오류 검출 코드가 들어갑니다.

### ④ 인코딩 (전기 신호 변환)

디지털 데이터(0과 1)를 랜선을 통해 흐를 수 있는 전기 신호(Voltage)로 바꾸어 송출합니다.

## 4. 실무에서의 구분 팁

1. **물리 서버 구축 시:** "서버에 **NIC**가 몇 포트짜리인가요?" 혹은 "**랜카드** 추가 장착 필요합니다"라고 말합니다.
    
2. **Windows 설정 시:** 아까 스크린샷에서 보셨듯이 OS 내부 설정에서는 "**네트워크 어댑터** 설정 변경"이라는 표현을 주로 봅니다.
    
3. **가상화(VMware 등) 시:** 물리적인 카드는 하나지만, 소프트웨어적으로 여러 개의 **가상 네트워크 어댑터(Virtual Adapter)**를 만들어 각 VM에 할당할 수 있습니다.
    

이제 `ipconfig` 명령어를 쳤을 때 나오는 '이더넷 어댑터'라는 항목이 어떤 물리적/논리적 구조를 거쳐 보이는 것인지 감이 오시나요?
