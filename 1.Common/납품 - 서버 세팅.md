### CD / DVD
이는 가상화 vKVM 을 통해 설치할 예정이다. ISO (가상 디스크) 연결용
1. 대시보드 가서 버추얼 콘솔 열기 > 버추얼미디어 > Disconnect Virtual Media
2. Media로 퉁쳐서 말하는데 ISO 파일로 압축을 할 수 있다.  
3. 가진 ISO 파일을 마운트한다.
4. Boot > Virtual CD/DVD 후 Power > Reset System (warm boot)
	**Boot 옵션에 대한 설명**
	- Graceful Shutdown : 
		- OS의 셧다운 명령어를 내린다. (천천히 절차대로)
			- 모든 데이터는 메모리에 올라간다. 거기에서 CPU (프로세서)가 데이터를 처리하고 디스크를 쓰게끔 되어있다. 메모리는 전기가 지속적으로 공급이 되어야 데이터를 저장할 수 있다. 공급이 끊어지면 다 날아간다.
			- DB는 트랜잭션 입출력 I/O는 메모리에 굉장히 많이 올라가기 때문에 정상 종료되지 않으면 메모리가 깨질 수도 있다.
		- 메모리 > 디스크로 데이터 옮긴 후 정상 종료
	- Power Off System : 강제종료. 전원 공급이 끊어지기 때문에 DB서버 등엔 문제 발생 가능
	- Reset System (warm boot): 
		- warm boot은 하드웨어/펌웨어 기본적으로 살려두고 운영체제만 재시작한다.
	- Power Cycle System (cold boot):
		- 펌웨어까지 한 번 내려갔다가 올라간다. 제조사마다 일부 차이는 있으나 h/w 자체를 처음부터 다시 전부 다 읽고, 전원공급 차단 후 다시 올리기 때문
		- 제조사마다 조금씩 옵션은 다르기때문에 추가 파악 필요할 수 있음
		- 간혹 우선순위 해놔도 디스크부터 부트되는 경우도 있다 (?)

5. Deploy OS > UEFI > OS 선택 > DVD_ROM (ISO) > Next .... >Finish
	- Unattended Install은 자동화. Boot Script 파일이 만들어져야만 자동배포가 가능하다. (이미 정의된 경우) = xml 등의 부트 파일이 별도로 존재한다.

> 가장 흔하게 겪는 상황을 예로 들어보겠습니다. 서버에 1TB 디스크를 장착하고
> Windows 운영체제를 설치하면, 디스크 용량이 1TB가 아니라 **약 931GB**로 표기됩니다.
> 1. **제조사가 제공한 1TB 물리 디스크:** 1,000,000,000,000 Byte를 담을 수 있습니다.
> 2. **Windows의 용량 인식:** 컴퓨터는 이 숫자를 1,024로 계속 나눕니다.
> - 1,000,000,000,000 / 1,024 / 1,024 / 1,024 = **약 931.32**
> 1. **가장 큰 문제점:** Windows는 계산은 2진수(GiB, TiB)로 해놓고, **화면에 보여줄 때는 관행적으로 'GB'나 'TB'라는 글자를 사용**합니다. (반면 macOS나 일부 리눅스는 10진수 기준인 1TB로 표기해주어 혼란이 적습니다.)
>  
> 결론적으로 디스크의 용량이 줄어들거나 불량인 것이 아니라, 제조사와 운영체제가 용량을 재는 **'자(Ruler)'의 눈금 기준이 달라서 생기는 표기의 차이**입니다. 인프라 시스템의 스토리지 용량을 산정(Sizing)할 때는 이 7% 정도의 손실(1TB 기준 931GB)을 반드시 계산에 넣어야 합니다.
