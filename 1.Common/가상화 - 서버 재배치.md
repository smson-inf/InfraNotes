# AD (Active Directory) 가입
## 개요 
가상화 서버 신규 등록 시 해당 사이트는 도메인 기반으로 움직임.
이러한 신규 등록 시 AD에 가입시켜줘야한다. (도메인에 연결)
AD는 가입된 다른 호스트들에 대해서 개별로 분리하여 정책 등을 줄 수 있음

## 절차
현 시점에서 6번 서버는 이미 DR에 존재하고, 네트워크 작업은 완료됨.
DNS에 AD1, AD2 IP 입력해준다.
IPV4 활성, IPV6은 비활성 해서 주로 기입
입력 후 대상 서버 재부팅 하면 적용 완료
최초 AD 가입 시 Computers에 나오게된다.
다른 가상 호스트들이 있는 그룹으로 옮겨준다. (부팅 된 후)
OU(Organization Unit) :
	- 큰 기업은 나라 별로 분류하거나, 더 작은 규모로 가면 부산지사, 서울지사 등으로 나누기도 한다. 부서 별 특징에 따라 정책을 줄 수 있다.
	- 원격 데스크톱이 특정 부분에는 안 붙게 하는 등의 제어가 가능

#### Local Repo
관리도구 같은 것처럼
8.8.8.8 외부 열려있으면 yum install로 저장소 가능하다. 다만 대부분 폐쇄망으로 가정하고, 레포를 잡아본다.


리눅스 로컬 레포지토리(Local Repository)는 ==외부 인터넷 연결 없이 서버나 PC 자체, 혹은 내부망(LAN)에 저장된 RPM 패키지(소프트웨어) 저장소==입니다. ISO 이미지나 로컬 디렉토리를 사용해 외부망 차단 환경(폐쇄망)에서도 의존성 문제없이 `yum`이나 `dnf`로 편리하게 패키지를 설치/업데이트할 수 있는 환경입니다. 

**주요 특징 및 장점:**

- **폐쇄망 환경 운영:** 인터넷이 안 되는 환경에서 패키지 설치 가능.
- **속도 향상:** 외부 인터넷이 아닌 로컬 저장소에서 다운로드하므로 설치 속도가 빠름.
- **의존성 해결:** 관련된 모든 패키지를 한곳에 모아두어 의존성 오류 발생을 줄임.
- **버전 관리:** 운영 환경에 맞는 특정 버전의 패키지만 유지/관리 가능. 

**주요 활용 방식:**

1. **ISO 파일 마운트:** 리눅스 설치 DVD/ISO 파일을 서버의 특정 디렉토리에 마운트하여 저장소로 사용.
2. **`createrepo` 활용:** 패키지(RPM)들을 디렉토리에 모아두고 `createrepo` 명령어를 사용하여 메타데이터를 생성하여 레포지토리로 등록. 

주로 내부망(Intranet)에 위치한 서버들이 보안 정책상 외부 패키지 서버에 접속할 수 없을 때 필수로 구성하는 방식입니다.

### LVM 설정

1. Hypver-V에서 디스크를 추가해줍시다.
2. 추가된 디스크는 sda가 있으니 sdb로 이름 짓습니다.
3. df -h 로 우선 sdb 존재 확인
4. vgscan (vgs) 으로 볼륨 그룹을 스캔합니다.
5. lsblk는 물리적 디스크를 볼 수 있습니다. (트리 구조)
6. fdisk -l 는 논리적으로 디스크를 볼 수 있습니다.
7. `vgcreate davabg /dev/sdb` 로 볼륨 그룹을 만듭니다.
8. 'pvs' 치면 붙어는 있지만 VG 안 잡혀있음. `lvs`는 로지컬 볼륨 확인 가능
9. `lvcreate -l +100%FREE -n datalv datavg`
	- +100%FREE는 100퍼 다 쓰겠다
	- -n datalv 는 로지컬볼륨 이름
	- 맨 뒤에는 선택할 로지컬 볼륨그룹
	- `lvcreate -L 10G -n datalv datavg` 이렇게 일부 용량만 만들수도 있다
10. `lvs` 하면 보임
11. `mkfs.xfs /dev/mapper/datavg-datalv`
	- LV는 생성 시 /dev/mapper 안에 들어간다 (연결성 있는 이름 참조)
	- xfs 파일 시스템으로 변경한다.
12. `vim /etc/fstab` 열고 
	/dev/mapper/datavg-datalv /data                 xfs     defaults        0 0
	입력하기

13. `cd /` 하고 'mkdir /data' 하면 데이터 파일 생성
14. `mount -a` 로 마운트 시킨다
15. `systemctl daemon-reload` 한 후 `df -h` 진행 시 마운트 정상 확인 가능


### LVM 확장 / 재설정

`lvremove /dev/mapper/datavg-datalv`
`lvcreate -L 10G -n datalv datavg`
`mount /data`
`lvextend -l +2G /dev/mapper/datavg-datalv` 한 후에 실제로 df -h에서 용량 늘리려면 `xfs_growfs /data` 하면 늘린 반영분 커짐
