# 개요
- 사이트 : 진주 경남국립대병원
- 작업 유형 : Cisco 가상화 서버 재배치 (3대 메인, 3대 DR)
- 필요 사항:
	- 네트워크 포설 (host5)
	- 잉여 서버 host6 설정, AD 가입
- 특이사항:
	- 자주 있는 작업은 아닌 편
	- DR-Replica 기능 사용하다가 스토리지 변경으로 인해 스토리지 간 Active 양방향 백업 기능을 사용하게 되면서, DR 서버로의 물리적 분리 및 양쪽 서버를 각각 활용하도록 하기 위한 작업
	- IBM의 특정 기능으로 2개의 스토리지 각각의 VM을 논리적으로 묶어 하나의 VM으로 인식하도록 제공하는 형태로

# AD (Active Directory) 가입
## 개요 
가상화 서버 신규 등록 시 해당 사이트는 도메인 기반으로 움직임.
이러한 신규 등록 시 AD에 가입시켜줘야한다. (도메인에 연결)
AD는 가입된 다른 호스트들에 대해서 개별로 분리하여 정책 등을 줄 수 있음

## 절차
현 시점에서 6번 서버는 이미 DR에 존재하고, 네트워크 작업은 완료됨.
DNS에 AD1, AD2 IP 입력해준다.
IPV4 활성, IPV6은 비활성 해서 주로 기입
입력 후 대상 서버 재부팅 하면 적용 완료
최초 AD 가입 시 Computers에 나오게된다.
다른 가상 호스트들이 있는 그룹으로 옮겨준다. (부팅 된 후)
OU(Organization Unit) :
	- 큰 기업은 나라 별로 분류하거나, 더 작은 규모로 가면 부산지사, 서울지사 등으로 나누기도 한다. 부서 별 특징에 따라 정책을 줄 수 있다.
	- 원격 데스크톱이 특정 부분에는 안 붙게 하는 등의 제어가 가능


## 연결 후 로그인
(domainroot) AD 계정으로 로그인 해야 가상 호스트들에 대한 접근, 통제 권한이 있게 된다.


> Windows 업데이트?
	자동재부팅 되지 않게 AD정책으로 막아둔다.
	윈도우는 업데이트 확인 버튼을 누르지않는 이상 실제로 윈도우 업데이트가 진행되진 않는다.
